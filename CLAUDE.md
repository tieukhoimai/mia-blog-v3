# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
yarn dev          # start dev server at localhost:3000
yarn build        # production build + postbuild generators
yarn serve        # serve production build
yarn lint         # lint app/, components/, lib/, layouts/, scripts/ with ESLint --fix
yarn analyze      # bundle analyzer build
```

No test suite is configured.

## Architecture

**Stack**: Next.js 14 App Router, React 18, Tailwind CSS 3, TypeScript, MDX via Contentlayer2 + Pliny. Package manager is Yarn 3 (Berry).

**Content pipeline**: MDX files in `data/blog/` and `data/authors/` are processed by Contentlayer2 at build time into typed TypeScript objects (`allBlogs`, `allAuthors` from `contentlayer/generated`). The `yarn build` command runs a postbuild step (`scripts/postbuild.mjs`) that:

1. Patches `.contentlayer/generated/index.mjs` to replace `assert { type: "json" }` with `with { type: "json" }` for Node.js v23+ compatibility
2. Generates `app/tag-data.json` (tag counts)
3. Generates `app/series-data.json` and `public/series-data.json` (series groupings)
4. Generates `public/search.json` (kbar search index)

These JSON files are **auto-generated** — do not edit them directly.

**Blog post frontmatter** (defined in `contentlayer.config.ts`): `title`, `date`, `tags`, `summary`, `image` (required), `draft`, `lastmod`, `layout`, `series`, `seriesOrder`, `authors`. Posts with a `series` field are grouped into article series; `seriesOrder` controls ordering within the series.

**Layouts**: Blog posts render with one of three layouts selected via the `layout` frontmatter field (default: `PostLayout`): `PostSimple`, `PostLayout`, `PostBanner` — all in `layouts/`.

**Routing**:

- `/blog` — paginated post list (`app/blog/`)
- `/blog/[...slug]` — individual post with dynamic layout selection
- `/series` — list of all series; `/series/[slug]` — series detail page (reads `app/series-data.json`)
- `/learning`, `/about`, `/resume`, `/tags/[tag]` — static pages

**Customization entry points**:

- Site metadata: `data/siteMetadata.js`
- Nav links: `data/headerNavLinks.ts`
- Author profile: `data/authors/default.mdx`
- Content Security Policy: `next.config.js` (update when adding analytics/comment providers)

**Environment variables** (optional, for Giscus comments and Google Analytics):

```
NEXT_PUBLIC_GOOGLE_ANALYTICS=G-XXXXXXXXXX
NEXT_PUBLIC_GISCUS_REPO=owner/repo
NEXT_PUBLIC_GISCUS_REPOSITORY_ID=
NEXT_PUBLIC_GISCUS_CATEGORY=
NEXT_PUBLIC_GISCUS_CATEGORY_ID=
```

**Husky pre-commit hook** runs lint-staged: ESLint on JS/TS files, Prettier on JS/TS/JSON/CSS/MD/MDX.

## Knowledge Graph

The homepage displays an interactive knowledge graph built from article co-occurrence.

- **Data**: `app/graph-data.json` and `public/graph-data.json` (both kept in sync)
- **Component**: `components/KnowledgeGraph.tsx` — client component, SVG-based, force-directed layout
- **Nodes**: tag slugs from `app/tag-data.json` with weight ≥ 2 (appear in 2+ articles)
- **Edges**: tag co-occurrence within the same article

**Run `/generate-graph`** after writing or editing any article in `data/blog/` to rebuild the graph. The hook in `.claude/settings.json` reminds you when a blog post is changed. `app/graph-data.json` is **not** auto-generated by `postbuild.mjs` — it requires an explicit `/generate-graph` run.
